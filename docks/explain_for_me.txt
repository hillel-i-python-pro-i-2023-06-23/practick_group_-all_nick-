папка .run:
    в нее создаем run.file через Run -> Edit Configurations -> тут настраиваем
папка app:
    в ней пишеться основной код нашего приложеня
папка venv:
    в ней делать нечего, просто окружение
.dockerignore:
    используется для указания файлов и директорий, которые должны быть проигнорированы при копировании контекста сборки в Docker-образ
.gitignore
    что бы не загружать в гитхаб все подряд
.pre-commit-config.yaml
    используется для настройки и определения хуков (hooks), которые будут выполняться перед коммитом в репозиторий Git.
docker-compose.override.dev.yml
    вы можете указать дополнительные сервисы, настройки окружения, привязки портов и другие параметры, которые применяются
    только во время разработки. Это позволяет вам настроить окружение для локальной разработки, отличное от продукционного
    окружения, чтобы облегчить разработку и отладку.
docker-compose.override.yml
    используется для переопределения или добавления конфигурации Docker Compose в вашем проекте. Он позволяет вам вносить
    изменения в основной файл docker-compose.yml, чтобы адаптировать его под конкретные требования разработки или среды выполнения.
docker-compose.yml
    вы определяете сервисы и их конфигурацию, включая образы Docker, привязки портов, переменные окружения, сети и другие параметры.
Dockerfile
    это текстовый файл, который содержит инструкции по созданию Docker-образа. В Dockerfile вы определяете, какие файлы,
     зависимости и команды нужно использовать для создания образа контейнера.
Makefile
    это текстовый файл, содержащий набор инструкций (целей) для автоматизации процесса сборки, тестирования и управления проектом.
README.md
    подключает в себя свойства makefile
requirements.txt
    для установки зависимостей, плагинов библиотек
run.py
    что бы запускать весь проэкт
setup.cfg
    надо для прописания свойств flake8
